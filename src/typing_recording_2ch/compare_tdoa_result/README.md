# タイプライター打鍵音識別におけるTDoA特徴量の評価

## メモ

TDoAの要素を入れたほうが若干精度が落ちた（1件違うのみのため誤差かもしれない）。  
理由として、TDoAの信頼度が低く、特徴量として意味をなしていないのだと思われる。  
改善するのであればTDoAの精度を上げることだが、ちょっと時間的制約でいったんこの結果で閉じる。  

次やるならば、新しいデータセットを取得し、TDoAありのモデルと無しのモデルで推論の正答率を確認し、今回のデータセット特有の結果ではないかを見る。  

## 概要

本資料は、タイプライターの打鍵音識別において、左右2chマイクから算出した **TDoA（Time Difference of Arrival：到達時間差）特徴量** が分類精度向上に寄与するかを確認した結果をまとめたものです。

実験では、以下の11キーを対象としました。

```text
E, H, I, L, O, P, R, SP, T, W, Y
```

サンプル数は合計 **1149件** です。

## 評価した特徴量

### 音響特徴量

従来の音響解析では、**CH1のみ**を使用します。

主な特徴量は以下です。

- MFCC
- MFCC delta
- RMS
- Zero Crossing Rate
- Spectral Centroid
- Spectral Bandwidth
- Spectral Rolloff
- Spectral Flatness
- FFT帯域エネルギー

特徴量数は **194個** です。

### TDoA特徴量

TDoAでは、**CH1とCH2の両方**を使用します。

追加した特徴量は以下の5個です。

| 特徴量 | 意味 |
|---|---|
| `tdoa_feature_ms` | 信頼できる場合のみ使うTDoA値。怪しい場合は0 |
| `tdoa_reliable` | TDoAを信用するかどうかの0/1フラグ |
| `best_corr` | CH1/CH2の相互相関の最大値。TDoAの信頼度目安 |
| `energy_CH1_over_CH2_dB` | TDoA計算窓内の左右エネルギー比 |
| `abs_tdoa_feature_ms` | TDoA絶対値。中央からのずれ量の目安 |

TDoAありの場合、特徴量数は **199個** です。

## 分類精度比較

| 条件 | 特徴量数 | CV accuracy | Test accuracy | 誤分類数 |
|---|---:|---:|---:|---:|
| 音響特徴量のみ | 194 | 0.9878 ± 0.0058 | 0.9870 | 3 |
| 音響特徴量 + TDoA | 199 | 0.9869 ± 0.0067 | 0.9826 | 4 |

## 分類結果の解釈

今回の結果では、**音響特徴量のみ**の方がわずかに高精度でした。

```text
音響特徴量のみ       : Test accuracy = 0.9870
音響特徴量 + TDoA    : Test accuracy = 0.9826
```

差は大きくありませんが、TDoAを追加しても精度向上は確認できませんでした。

そのため、現時点では以下の判断が妥当です。

```text
今回の分類モデルには、TDoA特徴量を入れない方がよい。
```

ただし、これは **TDoAが観測できなかった** という意味ではありません。

TDoAは左右位置を表す補助特徴として一部観測できていましたが、今回のキー識別ではCH1単独の音響特徴量が十分強く、TDoAを追加するメリットが出なかった、という結果です。

## TDoA信頼度の評価

TDoAの信頼度は、相互相関の最大値 `best_corr` を使って判定しました。

今回の判定条件は以下です。

```text
best_corr >= 0.45 → tdoa_reliable = 1
best_corr <  0.45 → tdoa_reliable = 0
```

つまり、`tdoa_reliable = 1` は「TDoA計算に使ったCH1/CH2の波形が比較的よく似ており、時間差として採用しやすい」という意味です。

ただし、これは **時間差が物理的に正しいことを保証する値ではありません**。  
あくまで、相互相関上の信頼度です。

## 全体のTDoA信頼度

全体では、TDoAが信頼可能と判定された割合は以下でした。

```text
tdoa_reliable = 1 : 862 / 1149
reliable率        : 75.0 %
```

全体としては **約75%** なので、TDoAがまったく取れていないわけではありません。

一方で、約25%のデータではTDoAを信用できない扱いになっています。  
分類特徴量としては、この不安定さがノイズになった可能性があります。

## キー別の傾向

信頼度が高かったキーは以下です。

```text
R, Y, T, E
```

これらは `tdoa_reliable` が90%前後であり、TDoA計算窓内のCH1/CH2波形が比較的よく対応していたと考えられます。

一方、信頼度が低めだったキーは以下です。

```text
P, SP, L, O
```

特に **P** は右側のキーであり、本来TDoAの効果が出てほしいキーですが、reliable率は約56%でした。  
この点から、TDoAが効いてほしい場面で安定しきれていない可能性があります。

## TDoAを入れても精度が上がらなかった理由

考えられる理由は以下です。

### 1. 音響特徴量だけで十分に分離できていた

CH1だけの音響特徴量で、すでにTest accuracyは約98.7%でした。  
この時点で分類器にとって必要な情報の大半は、音色・スペクトル・波形特徴に含まれていた可能性があります。

### 2. TDoAにキーごとのムラがあった

全体のreliable率は約75%でしたが、キー別では約56%〜94%までばらついていました。  
分類器から見ると、安定した特徴量というより、サンプルによって信用度が変わる特徴量だった可能性があります。

### 3. `best_corr` が高くても、位置情報として正しいとは限らない

`best_corr` は「CH1/CH2の波形がずらすと似ていた」ことを表します。  
しかし、戻り音には以下が混ざります。

- アームが戻る音
- キャリッジの動作音
- 筐体振動
- 反響
- マイクごとの音色差

そのため、相関が高くても、純粋な音源位置の到達時間差を表しているとは限りません。

## 現時点の結論

今回のデータセットでは、最終的な分類モデルとしては以下が妥当です。

```text
採用: 音響特徴量のみ
非採用: TDoA特徴量
```

ただし、TDoAの検証自体は有意義でした。

```text
TDoAは一部のキーで観測可能だった。
しかし、今回のSVM分類では精度向上には寄与しなかった。
```
