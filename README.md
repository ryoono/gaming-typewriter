# GamingTypewriter

タイプライターの打鍵音から、押されたキーを推定するプロジェクトです。
タイプライターを「最高の打鍵音と打鍵感を持つキーボード」として再解釈し、実機のタイプライター操作によってPC上に文字を入力することを目指します。

## 概要

本プロジェクトでは、タイプライターの各キーを打鍵した際に発生する音を録音し、その音響特徴を機械学習によって分類することで、どのキーが押されたかを推定します。

一般的なPCキーボードとは異なり、タイプライターはキー入力を電気信号として直接取得できません。そこで、打鍵時に発生する音と機械的な振動を利用し、非侵襲的にキー入力を推定します。

最終的には、タイプライターを操作するとPC上に対応する文字が表示される「ゲーミングタイプライター」の実現を目標としています。

## 背景

近年、メカニカルキーボードや自作キーボードの分野では、打鍵感や打鍵音にこだわる文化が広がっています。スイッチの種類、キーキャップ、ケース構造、吸音材などを工夫し、自分好みの入力体験を追求するユーザーも多く存在します。

一方で、タイプライターは機械式入力装置として非常に強い個性を持っています。

* 金属機構による明確な打鍵感
* 紙に印字するための力強い打鍵音
* キャリッジが動く機械的なフィードバック
* レトロで物理的な操作体験

これらは現代のキーボードにはない魅力です。

本プロジェクトでは、タイプライターそのものを改造して電気接点を追加するのではなく、打鍵音を解析することでキー入力を推定します。これにより、タイプライター本体への大きな改造を避けながら、タイプライターをPC入力装置として活用することを目指します。

## 目的

本プロジェクトの目的は、タイプライターの打鍵音を用いて押されたキーを識別し、PC上で文字入力として扱える仕組みを構築することです。

主な目的は以下の通りです。

1. タイプライターの各キーに固有の音響特徴が存在するかを検証する
2. 打鍵音から押下キーを分類する機械学習モデルを作成する
3. 加速度センサを用いて打鍵タイミングを検出し、録音データを自動で切り出す
4. 推定結果をPC上の文字入力へ変換する
5. タイプライターを「ゲーミングタイプライター」として利用できるシステムを構築する

## 環境

Python 3.13.0  
オーディオインターフェース：ZOOM AMS-44  

## システム構成

本システムは、以下の要素で構成されます。

```text
タイプライター
  ├─ 打鍵音
  │    └─ マイクで録音
  │
  └─ キャリッジの動き
       └─ 加速度センサで検出

録音・解析用PC
  ├─ 加速度センサの反応をトリガとして検出
  ├─ 打鍵前後100ms、合計200msの音声データを切り出し
  ├─ 音響特徴量を抽出
  ├─ 機械学習モデルでキー分類
  └─ 推定キーをPC上に入力
```

## 手法

### 1. 打鍵音の録音

タイプライターのキーを押した際に発生する音をマイクで録音します。

キーごとに、以下のような違いが音に表れる可能性があります。

* タイプバーの位置による到達時間差
* 打鍵時の金属音の違い
* キーごとの機械的な動作経路の違い
* キャリッジや筐体への振動伝達の違い
* 印字位置やアーム長の違いによる周波数成分の差

これらの違いを機械学習モデルに学習させることで、キー分類を行います。

### 2. 加速度センサによる打鍵タイミング検出

タイプライターのキャリッジ部分に加速度センサを設置します。

キーが押されると、印字動作に伴ってキャリッジが左方向へ移動します。この動きを加速度センサで検出し、打鍵が発生したタイミングを取得します。

加速度センサを使う理由は、音声だけで打鍵開始点を検出するよりも、録音データの切り出しタイミングを安定させやすいためです。

### 3. 音声データの自動切り出し

加速度センサに反応があった時刻を基準として、前後100ms、合計200msの音声データを切り出します。

```text
打鍵検出時刻 = t

切り出し範囲:
t - 100ms 〜 t + 100ms

合計: 200ms
```

この範囲には、以下のような情報が含まれることを想定しています。

* キー押下直後の機械音
* タイプバーが動作する音
* 活字が紙やプラテンに衝突する音
* キャリッジが移動する音
* 打鍵後の残響や振動

### 4. データ収集

全てのキーについて、同じ手順で録音データを収集します。

データ収集時には、操作者が現在打鍵するキーのラベルを指定し、そのキーを複数回打鍵します。加速度センサが反応するたびに、対応する200msの音声データを保存します。

保存データには、以下の情報を含める想定です。

* キーラベル
* 録音日時
* サンプリングレート
* 音声データ
* 加速度センサの検出時刻
* 収録条件
* 試行番号

例:

```text
dataset/
  A/
    A_0001.wav
    A_0002.wav
    A_0003.wav
  B/
    B_0001.wav
    B_0002.wav
    B_0003.wav
  C/
    C_0001.wav
    C_0002.wav
    C_0003.wav
```

### 5. 特徴量抽出

切り出した200msの音声データから、分類に使用する特徴量を抽出します。

使用する特徴量の候補は以下です。

* 波形データそのもの
* メルスペクトログラム
* MFCC
* ゼロ交差率
* スペクトル重心
* スペクトル帯域幅
* RMSエネルギー
* ピーク位置
* 複数マイクを使用する場合の到達時間差

初期段階では、MFCCやメルスペクトログラムを用いた分類を検討します。十分なデータが得られる場合は、CNNなどを用いた画像分類的なアプローチも検討します。

### 6. 学習モデルの作成

全キーの200ms音声データを用いて、押されたキーを分類するモデルを作成します。

分類対象は、タイプライター上の各キーです。

```text
入力: 200msの打鍵音データ
出力: 推定されたキーラベル
```

モデルの候補は以下です。

* k近傍法
* サポートベクターマシン
* ランダムフォレスト
* ロジスティック回帰
* MLP
* CNN

初期実装では、比較的少量のデータでも扱いやすいモデルから試し、分類精度を確認します。その後、必要に応じてニューラルネットワーク系のモデルへ拡張します。

## データ収集フロー

```text
1. 打鍵対象のキーを選択する
2. 操作者が指定されたキーを打鍵する
3. 加速度センサがキャリッジの動きを検出する
4. 検出時刻の前後100msの音声を切り出す
5. キーラベル付きの音声データとして保存する
6. 全てのキーについて繰り返す
7. 収集したデータで分類モデルを学習する
```

## 推論フロー

```text
1. タイプライターを打鍵する
2. 加速度センサが打鍵タイミングを検出する
3. 前後100msの音声データを切り出す
4. 学習済みモデルに入力する
5. 押されたキーを推定する
6. 推定結果をPC上に文字として表示する
```

## 評価方法

モデルの性能評価には、以下の指標を使用します。

* Accuracy
* Precision
* Recall
* F1-score
* Confusion Matrix

特に、どのキー同士が誤分類されやすいかを確認するため、混同行列を重視します。

タイプライターの構造上、近い位置にあるキーや似た機械音を持つキーは誤分類されやすい可能性があります。そのため、単純な正解率だけでなく、誤分類の傾向分析も行います。

## 課題

現時点で想定している課題は以下です。

* キーごとの音響差が十分に分類可能か
* 打鍵の強さや速度によるばらつき
* 操作者の癖による音の違い
* 周囲の環境音の影響
* マイク位置の変化による特徴量の変化
* 複数キーの音が似ている場合の識別
* 実時間推論時の遅延
* Shiftキーや改行、スペースなどの特殊キーの扱い
